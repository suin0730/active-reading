# [Item 51] 메서드 시그니처를 신중히 설계하라

API 설계 요령
- 메서드 이름 신중하게 짓기
  - 표준 명명 규칙을 따라야 한다.
  - 같은 패키지에 속한 다른 이름과 일관되게 지어야 한다.
  - 개발자 커뮤니티에서 널리 받아들여지는 이름을 사용하라.
  - 너무 긴 이름은 피하라. (자바 라이브러리 API 가이드 참고)
- 편의 메서드를 너무 많이 만들지 말기
  - 메서드가 너무 많은 클래스는 사용하고 유지보수하기 어렵다.
  - 클래스, 인터페이스는 자신의 기능을 완벽히 수행하도록 제공해야 한다.
- 매개변수 목록은 짧게 유지하기
  - 개수가 4개 넘어가면 기억하기 어렵다.
  - 같은 타입 매개변수가 여러 개 나오면 혼란을 줄 수 있기 때문에 피해야 한다.
  - 짧게 만드는 방법
    - 여러 메서드로 쪼개 기능을 원자적으로 제공한다.
    - 매개변수 여러 개를 묶는 도우미 클래스를 만든다.
    - 매개변수 일부를 생략해도 괜찮은 때는, 빌더 패턴을 응용한다.
- 매개변수 타입으로 인터페이스 사용하기
  - 인터페이스를 사용하면 클라이언트가 인자를 유연하게 넘길 수 있게 한다.
- boolean보다는 원소 2개짜리 열거 타입 사용하기
  - 메서드가 하는 일을 명확하게 알려준다.
  - 클래스가 확장되는 경우에 강하다.
